## あなたの外観 {#appearance}

{{< figure src="assets/profile-card.png" width="75%" caption="プロフィールカードでは表示名、アイコン、ヘッダー画像、プロフィールが表示される" >}}

**設定の「プロフィール」-「プロフィールを編集」**に移動し、あなたのプロフィールが他の人にどのように見られるかを変更できます。

### 表示名 {#name}

表示名（＝ディスプレイネーム）は他のユーザーにとって、あなたのアドレスの前に表示される名前です。標準のMastodonでは30文字までを表示名として設定できます。

### プロフィール {#bio}

プロフィール（＝バイオ）は、プロフィール上でノートとして表示される、あなた自身の短い説明です。標準のMastodonでは500文字までの文章を記述できます。すべてのリンクは、実際の長さに関係なく23文字としてカウントされます。そのため、文字数を節約するためにリンク短縮サービスを使用する必要はありません。事実、リンク短縮サービスの使用は強く推奨されていません。

### アイコン {#avatar}

アイコン（＝アバター）は投稿の横に表示され、あなたであることを示す視覚的な情報です。サイズが最大2MBまでのPNG、GIF、またはJPG形式の画像をアイコンとしてアップロードできます。この画像は400x400に縮小されます。

### ヘッダー {#header}

ヘッダーは、プロフィールの上部で表示されるバナー画像です。フォローリストやディレクトリでプロフィールカードが使われる場合にはこのヘッダーも表示されます。サイズが最大2MBまでのPNG、GIF、またはJPG形式の画像をヘッダーとしてアップロードできます。この画像は1500x500に縮小されます。

## プロフィールに関するフラグ {#flags}

プロフィールに関するフラグを設定することで、どのようにマストドンを使っているかを、他の人に知らせることができます。

{{< figure src="assets/bot-flag.png" width="50%" caption="自動で操作していることを知らせるマークを表示しているプロフィールの外観" >}}

### 鍵アカウント {#locked}

設定の「プライバシーとつながりやすさ」にある「フォローリクエストを不要にする」のチェックを外して、鍵アカウントにすることで、次の2つが引き起こされます。

- 新しいフォロワーを自動的に受け入れなくなります。そのため、その新しいフォロワーを手動で承認する必要があります。
- プロフィールに鍵アイコンを表示し、他のユーザーに対してフォローがすぐに受け入れられないことを知らせます。

### BOTアカウント {#bot}

設定の「プロフィール」-「プロフィールを編集」にあるBOT（ボット）のフラグを有効にすると、BOTアイコンがプロフィールに追加されます。BOTアイコンは、そのプロフィールが自動で行動したり、そのアカウントを持っている人に監視されていない可能性があったりすることを他の人に知らせます。他のソフトウェアではBOTプロフィールを普通のプロフィールと異なる扱いにしているかもしれませんが、Mastodonでは現在、視覚的な表示としてのみBOTフラグを取り扱っています。

### ディレクトリ {#discoverable}


設定の「プライバシーとつながりやすさ」にある「アカウントを見つけやすくする」にチェックを入れることで、あなたのプロフィールがディレクトリ（＝プロフィールディレクトリ）で、あなたのプロフィールが見つけられるようになります。

訳注：Mastodonのページでは基本的に、ドメイン名が記載されている横にあるリンクの一つに「ディレクトリ」があります。

訳注：「アカウントを見つけやすくする」の設定は、トレンドでの表示などにも影響します。

## プロフィール補足情報 {#fields}

プロフィール補足情報（＝エキストラフィールド）は、プロフィール上で簡単に拾い読みできる補足情報を追加する手段です。4つの行を持っていて、そこではラベルとその値を記述できます。例えば、次のように記入できます。

| ラベル | 内容 |
| :--- | :--- |
| 年齢 | 25歳 |
| 国 | ドイツ |
| 代名詞 | he/him |
| ウェブサイト | https://example.com |

何を書くかは完全にあなた次第です。その内容はメンション、ハッシュタグ、カスタム絵文字、リンクを含められます。

### リンク検証 {#verification}

申請書による検証と青いチェックマークへの対応は、それを処理する団体が中央型で権威を持つことが不可欠です。しかしMastodonでは、プロフィールに書いたリンクが本当にあなたのものであることを証明するために、リンク先のページからもMastodonのプロフィールにリンクすることでそのことを参照できます。Mastodonからのリンクの一つがすでに知られていて信用できるあなた個人のホームページである場合には、書類を使った本人確認の次善策として有効な手段となるでしょう。

{{< hint style="info" >}}
Mastodonは自分で構築できるため、人々がすでに信頼しているあなたのドメインでMastodonをホストすること以上に、あなたの本人確認を行う良い手段はありません。
{{< /hint >}}

あなたはプロフィール補足情報にHTTPSリンクを書くと、Mastodonは、そのリンク先のページで`rel="me"`属性をもったMastodonのプロフィールへのリンクがあるかを確認します。もし確認できれば、プロフィール補足情報に書いたリンクの横に検証済みのチェックマークが付きます。これにより、あなたがそのリンクを所有していることを示せます。そしてMastodonは、プロフィール補足情報内のリンクにも`rel="me"`を設定します。なお、自分のウェブサイトにリンクしている場合は、そのウェブサイトでMastodonプロフィールへのリンクを次のように記述することになります。

```html
<a href="https://social.example.com/@username" rel="me">Follow me on Mastodon!</a>
```

ウェブサイトのhead要素に次のように直接埋め込むこともできます。

```html
<link href="https://social.example.com/@username" rel="me">
```

訳注：設定の「プロフィール」-「認証」に記載の情報も参考にしてください。

#### リンク確認のための検証条件

Mastodonでは、プロフィール補足情報の「確認済みリンク」を次の条件で検証します。


- プロフィール補足情報の値は**HTTPSで始まるリンクである必要があります**。そうではない値は処理されません。その他のリンク形式も処理されません。特に、暗号化されないHTTPのリンクは安全ではなく、転送中に攻撃者によって改竄される可能性があるため、処理されません。
- バージョン4.0からは、**ホストネームはIDN（国際化ドメイン名）正規化の後で変更できません**。これは、いくつかのUnicode文字がASCII文字と紛らわしいほど似ていることを利用して、文字列でそうした文字を混在させておこなわれるホモグラフ（homograph）攻撃を防ぐためです。たとえば、`U+0061`（英語の小文字の「a」）と`U+0430`（キリル文字の小文字の「а」）はほとんど同じに見えますが、まったく異なる文字です。この見た目の類似性は、人々を誤解させるために悪意を持って使われる可能性があります。


Mastodonはそのリンクを解決して、その場所にあるウェブページを取得し、ソース内に次の基準に合致するリンクがあるかを探します。

- その解決されたページには、`rel="me"`属性をもった`a`タグか`link`タグが少なくとも1つ含まれていなければなりません。
  - それらの要素の1つでの`href`属性は、あなたのMastodonプロフィールのURLに一致している必要があります。
- `rel="me"`をもったリンクが見つからなかった場合、Mastodonは*最初の*リンクを探すので、その`href`値があなたのMastodonプロフィールのURLにリダイレクトしている必要があります（これは、リンク短縮サービスを使っていて`rel="me"`が使われていないウェブページへの限定的な対応です）。
- 1MBを超えるHTTPレスポンスは処理されません。

**このリンクは`iframe`内に書かれていることを許容しません。**`iframe`内にこのリンクがあったとしたら、実質的に同じウェブページ上のものではなく、現在のページに埋め込まれている外部のウェブページ上のものであることを意味するからです。（注：一部の「ブロックベース」のCMSソフトウェアは、ブロック要素をiframe内に包含することがあり、この理由で検証ができなくなります。）

{{< hint style="info" >}}
あなたのウェブページに`rel="me"`リンクを追加した*後で*、Mastodonのプロフィール補足情報を保存するようにしてください！ 検証プロセスはプロフィールを保存したときに開始されますが、完了するまでに時間がかかる場合があります。もし`rel="me"`リンクを追加しても検証できない場合は、プロフィール上でそのリンクを削除して保存し、再度そのリンクを追加して保存し直してみてください。
{{< /hint >}}
