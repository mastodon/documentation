## 文本 {#text}

每条嘟文更新的主体内容可以使用文本字段进行编辑。默认的字符限制为 500 个字符。

### 链接 {#links}

{{< figure src="assets/compose-links.png" width="50%" caption="链接必须以 http(s):// 开头，无论长度如何，都计为 23 个字符" >}}

如果你在嘟文中包含链接，它们必须以 `http://` 或 `https://` 开头。所有链接都计为 23 个字符，无论实际长度如何，因此无需使用链接缩短器来节省字符。实际上，我们强烈建议**不要**使用链接缩短器。

### 提及 {#mentions}

{{< figure src="assets/compose-mentions.png" width="50%" caption="针对本站和外站用户的提及建议" >}}

你可以输入用户的完整地址来提及用户，例如 `@alice@example.com`。请注意，任何使用 `@word` 的形式都将被解释为提及用户名为 `word` 的本站用户（如果该用户存在）。只有用户名部分会计入字符限制，域名不计入。

### 话题标签 {#hashtags}

{{< figure src="assets/compose-hashtags.png" width="50%" caption="根据使用频率自动建议话题标签" >}}

你可以使用 `#话题标签` 使你的嘟文可以被搜索该话题标签的任何人发现。话题标签可以包含字母、数字和下划线，但不能只包含数字。

### 自定义表情 {#emoji}

{{< figure src="assets/compose-custom-emoji.png" width="50%" caption="选择器中提供了一系列自定义表情" >}}

每个站点都提供一组你可以使用的自定义表情，类似于 Slack 或 Discord。 你可以使用表情的短代码（如 `:thinking:`）或点击撰写框中的表情，然后在“自定义”类别中浏览来使用表情。 你还可以浏览和搜索标准的 Unicode 表情符号。

## 附件 {#attachments}

你可以将文件或投票附加到你的嘟文。

### 文件 {#media}

{{< figure src="assets/compose-media-attachment.png" width="50%" caption="带有媒体附件缩略图的新嘟文，并带有用于其他选项的“编辑”按钮" >}}

点击照片集图标，就可以将文件附加到你的嘟文。 你可以附加以下内容：

* **图片** (PNG, JPG, HEIF, WEBP, AVIF) **最大 16MB**。 图片将被缩放到 830 万像素（足够用于发布 3840x2160 图像）。 每条嘟文最多可以附加 4 张图片。
* **动态 GIF** (GIFV) **最大 16MB** 会被转换为无声 MP4。 GIF 尺寸必须小于 1MP (1280x720)。 你还可以上传无声 MP4 和 WebM 文件，这些文件将在客户端中像动态 GIF 一样循环播放。 每条嘟文只能附加一个动态 GIF。
	* 静态 GIF 文件与动态 GIF 文件适用相同的限制。 如果你在上传这些图像时遇到问题，则需要将其转换为另一种支持的图像格式。
* **视频** (MP4, M4V, MOV, WebM) **最大 99MB**。 视频将被转码为 H.264 MP4，最大比特率为 1300kbps，帧率为 120fps。 每条嘟文只能附加一个视频。
* **音频** (MP3, OGG, WAV, FLAC, OPUS, AAC, M4A, 3GP) **最大 99MB**。 音频将使用 V2 VBR（大约 192kbps）转码为 MP3。 每条嘟文只能附加一个音频文件。

#### 编辑媒体 {#edit}

{{< figure src="assets/edit-media.png" caption="编辑媒体以添加媒体描述（替代文本）或选择预览缩略图的焦点" >}}

单击附件缩略图上的“编辑”按钮，将弹出一个对话框，你可以在其中添加媒体描述或更改焦点。 虽然添加媒体描述是可选的，但最好添加媒体描述，简要描述媒体中包含的内容。 当媒体由于任何原因未能加载，或者被屏幕阅读器和其他辅助技术访问时，这些描述就将显示。 设置焦点也是可选的，但是当预览缩略图未以 16:9 的宽高比显示时，可以使其看起来更好。

### 投票 {#polls}

{{< figure src="assets/compose-polls.png" width="50%" caption="新嘟文，其中包含“单选”样式的投票，该投票在一天后到期" >}}

单击条形图图标以将投票附加到你的嘟文。

* 你最多可以添加 4 个选项。 每个选项最多可以包含 25 个字符。
* 投票默认为单选。 点击单选按钮，将投票切换为多选复选框。
* 投票可以设置为在 5 分钟、30 分钟、1 小时、6 小时、1 天、3 天或 7 天后过期。

## 发布级别 {#privacy}

{{< figure src="assets/post-visibility.png" width="50%" caption="新嘟文，可以选择修改发布级别" >}}

| 级别 | 公共时间线 | 永久链接 | 账户视图 | 首页信息流 |
| :--- | :--- | :--- | :--- | :--- |
| 公开 | 是 | 是 | 是 | 是 |
| 悄悄公开 | 否 | 是 | 是 | 是 |
| 关注者 | 否 | 在同一站点上登录 | 应用内或登录 | 是 |
| 特定的人 | 否 | 登录并被提及 | 应用内或登录 | 否 |

嘟文可以使用以下四种不同的隐私级别发布，如下所述。 你可以[为你的嘟文设置默认隐私级别](../preferences#posting)，并且你可以通过选择嘟文文本下方的“更改嘟文隐私设置”图标来更改草稿嘟文的隐私级别。

### 公开 {#public}

默认选项。

* 任何人无需登录即可在通过访问永久链接看到你的嘟文。
* 你的嘟文将出现在应用内的公共时间线中。
* 你的关注者将在其首页信息流中收到嘟文，并且任何被提及的人都将收到通知。
* 你的嘟文可以被转发到其他首页信息流中。

### 悄悄公开 {#unlisted}

与公开完全相同，但有以下区别：

* 你的嘟文不会出现在 Mastodon 的实时信息流或探索功能中。
* 即使你的帐户已选择加入全文搜索，你的嘟文也不会出现在搜索结果中。

此选项以前称为“未列出”。

### 关注者 {#followers}

一种更为限制的传递选项。

* 通过永久链接查看你的嘟文需要与关注你或被提及的用户在同一站点登录。
* 除你的关注者和任何被提及的人之外，你的嘟文不会出现在应用内的其它地方。
* 你的关注者将在其首页信息流中收到嘟文，并且任何被提及的人都将收到通知。
* 你的嘟文不能被转发，除非要转发嘟文的使你自己。

{{< hint style="warning" >}}
要有效地发布私人（仅限关注者）嘟文，你必须**锁定你的帐户**——否则，任何人都可以关注你以查看较旧的嘟文。
{{< /hint >}}

{{< hint style="danger" >}}
请注意，Mastodon 上的嘟文隐私是按嘟文设置的，而不是帐户范围的，因此**无法将过去的公开嘟文设置为私有。**
{{< /hint >}}

### 特定的人 {#private}

仅将你的嘟文发送给被提及的用户。

* 通过永久链接查看你的嘟文需要被提及的人在同一站点登录。
* 除被提及的人之外，你的嘟文不会出现在应用内的其它地方。
* 所有被提及的人都将收到通知。但嘟文不会出现在首页时间线中。
* 你的嘟文不能被转发。

{{< hint style="warning" >}}
**不要通过私人提及分享危险和敏感信息**。 Mastodon 不是像 Signal 或 Matrix 这样的加密消息应用程序，发送者和接收者所在站点的数据库管理员理论上可以访问文本。
{{< /hint >}}

## 回复的可见性 {#replies}

只有当关注者同时关注你和你回复的人时，你的回复才会出现在关注者的首页信息流中。

关于这一点的一些说明：

* 你发布的以 `@mention` 开头的新嘟文**不会**被视为回复，因此它将显示在你所有关注者的首页信息流中。
* 回复的可见性（公开 vs 未公开 vs 仅限关注者）**不会**影响其是否显示在首页信息流中。
* 如果你的嘟文是公开的，且包含关注的话题标签，则无论它是否是回复，都会显示在首页信息流中。
* 其他 Fediverse 软件可能会在首页信息流中显示你的所有回复，无论这些嘟文回复的是谁。

## 内容警告 {#cw}

{{< figure src="assets/status-cw.png" width="50%" caption="展开的嘟文，带有内容警告，并且附加的图像也被隐藏" >}}

Mastodon 提供的一项你在其他社交网络上可能没有看到的功能是，你可以选择向你的嘟文附加“内容警告”。 当包含内容警告时，嘟文内容默认会被折叠，并且仅显示内容警告，类似于电子邮件的标题或“阅读更多”的分割线。 这可以用来为你的嘟文添加摘要或主题，折叠长嘟文，或者为嘟文的正文提供其他上下文或设置。

嘟文附加了媒体时，会出现一个复选框，允许你“将媒体标记为敏感”。 默认情况下，这会将全部媒体隐藏在模糊的缩略图后面。 将内容警告添加到嘟文也会自动将媒体标记为敏感。
